# Monitorización
- En Windows, abre el Visor de Eventos. Ve a Registros de Windows > Sistema. Filtra el registro actual para mostrar solo los eventos con el nivel "Error" y "Crítico" de las últimas 24 horas. Investiga uno de los errores que encuentres, copiando el "Origen" y el "ID del evento" y buscándolos en Google para entender su significado.
  - https://www.jesusninoc.com/02/07/ejercicios-de-powershell-ejercicios-con-fechas/
```PowerShell
$desde = (Get-Date).AddHours(-24)

Get-WinEvent -LogName System |
Where-Object {
    $_.TimeCreated -ge $desde -and
    ($_.LevelDisplayName -eq 'Error' -or $_.LevelDisplayName -eq 'Critical')
} |
Select-Object -First 20 TimeCreated, LevelDisplayName, ProviderName, Id |
Format-Table -AutoSize
```

- Comandos y cmdlets
  - Monitorización en Linux

| Comando | Qué monitoriza | Uso básico | Qué muestra |
|---|---|---|---|
| `top` | CPU, memoria, procesos | `top` | Procesos en tiempo real, uso de CPU y RAM |
| `htop` | CPU, memoria, procesos | `htop` | Versión más visual e interactiva de `top` |
| `ps` | Procesos | `ps aux` | Lista completa de procesos |

  - Monitorización en PowerShell

| Comando | Qué monitoriza | Uso básico | Qué muestra |
|---|---|---|---|
| `Get-Process` | Procesos | `Get-Process` | Lista de procesos activos |
| `Get-Process \| Sort-Object CPU -Descending \| Select-Object -First 10` | Procesos por CPU | `Get-Process \| Sort-Object CPU -Descending \| Select-Object -First 10` | Procesos que más CPU consumen |

- Ejercicios
```Bash
#!/bin/bash

echo "=============================="
echo " MONITORIZACIÓN BÁSICA LINUX "
echo "=============================="
echo

# Fecha y hora
echo "Fecha: $(date)"
echo

# Procesos
echo "Número total de procesos:"
ps -e --no-headers | wc -l
echo

# Primeros 5 procesos por uso de CPU
echo "Top 5 procesos por CPU:"
ps -eo pid,comm,%cpu,%mem --sort=-%cpu | head -n 6
echo

echo "Fin del informe."
```
