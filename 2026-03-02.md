# Monitorización
- En Windows, abre el Visor de Eventos. Ve a Registros de Windows > Sistema. Filtra el registro actual para mostrar solo los eventos con el nivel "Error" y "Crítico" de las últimas 24 horas. Investiga uno de los errores que encuentres, copiando el "Origen" y el "ID del evento" y buscándolos en Google para entender su significado.
- Explorador de procesos https://learn.microsoft.com/es-es/sysinternals/downloads/process-explorer
- Sysmon https://fwhibbit.es/sysmon-el-gran-hermano-de-windows-y-el-super-sysmonview
- Comandos y cmdlets
  - Monitorización en Linux

| Comando | Qué monitoriza | Uso básico | Qué muestra |
|---|---|---|---|
| `top` | CPU, memoria, procesos | `top` | Procesos en tiempo real, uso de CPU y RAM |
| `htop` | CPU, memoria, procesos | `htop` | Versión más visual e interactiva de `top` |
| `ps` | Procesos | `ps aux` | Lista completa de procesos |
| `free` | Memoria RAM y swap | `free -h` | Memoria usada, libre y swap |
| `vmstat` | CPU, memoria, procesos, I/O | `vmstat 1` | Estadísticas del sistema cada segundo |
| `iostat` | Disco y CPU | `iostat -xz 1` | Uso de disco, lecturas, escrituras, espera |
| `iotop` | I/O por proceso | `sudo iotop` | Qué procesos están usando más disco |
| `df` | Espacio en disco | `df -h` | Espacio usado y libre por sistema de archivos |
| `du` | Uso de disco por carpeta | `du -sh /ruta` | Tamaño de un directorio |
| `sar` | Rendimiento histórico | `sar -u 1 5` | Estadísticas de CPU, RAM, red y más |
| `mpstat` | CPU por núcleo | `mpstat -P ALL 1` | Uso de cada núcleo |
| `pidstat` | Uso de recursos por proceso | `pidstat 1` | CPU, memoria e I/O por PID |
| `ss` | Red y sockets | `ss -tulnp` | Puertos abiertos y conexiones |
| `iftop` | Tráfico de red | `sudo iftop` | Consumo de red por conexión |
| `journalctl` | Logs del sistema | `journalctl -xe` | Registros de systemd |
| `uptime` | Carga del sistema | `uptime` | Tiempo encendido y carga media |

- Ejercicios
```Bash
#!/bin/bash

echo "=============================="
echo " MONITORIZACIÓN BÁSICA LINUX "
echo "=============================="
echo

# Fecha y hora
echo "Fecha: $(date)"
echo

# Hostname
echo "Equipo: $(cat /etc/hostname 2>/dev/null)"
echo

# Uptime
echo "Tiempo encendido:"
cat /proc/uptime | cut -d' ' -f1 | awk '{print $1 " segundos"}'
echo

# Carga media
echo "Carga media (1, 5 y 15 min):"
cat /proc/loadavg | cut -d' ' -f1-3
echo

# CPU (modelo)
echo "CPU:"
grep "model name" /proc/cpuinfo | head -n 1 | cut -d':' -f2-
echo

# Número de CPUs
echo "Núcleos detectados:"
grep -c "^processor" /proc/cpuinfo
echo

# Memoria
echo "Memoria (kB):"
grep -E "MemTotal|MemFree|MemAvailable|SwapTotal|SwapFree" /proc/meminfo
echo

# Procesos
echo "Número total de procesos:"
ps -e --no-headers | wc -l
echo

# Primeros 5 procesos por uso de CPU
echo "Top 5 procesos por CPU:"
ps -eo pid,comm,%cpu,%mem --sort=-%cpu | head -n 6
echo

# Uso de disco
echo "Uso de disco:"
df -h / | tail -n 1
echo

# Interfaces de red
echo "Interfaces de red detectadas:"
cat /proc/net/dev | tail -n +3 | cut -d':' -f1 | tr -d ' '
echo

# Tráfico de red (bytes recibidos y enviados)
echo "Tráfico de red:"
cat /proc/net/dev | tail -n +3 | while IFS= read -r linea; do
    interfaz=$(echo "$linea" | cut -d':' -f1 | tr -d ' ')
    datos=$(echo "$linea" | cut -d':' -f2)
    rx=$(echo "$datos" | awk '{print $1}')
    tx=$(echo "$datos" | awk '{print $9}')
    echo " - $interfaz | RX: $rx bytes | TX: $tx bytes"
done
echo

# Usuarios conectados
echo "Usuarios conectados:"
who
echo

echo "Fin del informe."
```

# Resolución incidencias
- Repaso de AD (GPO).
- En un Windows Server, instala el rol de WSUS. Configúralo para que sincronice las actualizaciones de Windows 10. Luego, crea una GPO y configúrala para que los clientes de tu dominio apunten a este servidor WSUS para buscar actualizaciones. Aplica la GPO a una OU con equipos cliente y fuerza la detección de actualizaciones en uno de ellos.
- En una máquina virtual con Windows, habilita el Escritorio Remoto. Desde tu PC anfitrión, abre el cliente de Conexión a Escritorio Remoto (mstsc.exe) e introduce la dirección IP de la máquina virtual para conectarte a ella. Experimenta la sensación de controlar la VM como si estuvieras sentado delante de ella.
